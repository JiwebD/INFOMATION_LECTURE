라우팅 프로토콜

정적
관리자가 직접 목적지에 대한 방향을 정한다. 
=> 지정된 방향으로만 가면되니까 속도가빠름 
=> 부하적음 => 네트워크 변화에 민감하게 대응하지 못한다.

static -> 단일 목적지에 대한 방향을 관리자가 직접 넣어줌
default -> 지정하지 않은 모든 목적지에 대한 방향을 관리자가 직접 넣어줌


동적 : 네트워크 전적으로 학습하여 최적화 및 자동화
=> 네트워크 변화에 민감하게 대응할 수 있다.

디폴트 라우팅

하나의 주소가 아닌 모든 주소로 설정하는 것
= 0.0.0.0

가장 자리 라우터에는
static 경로를 정해줄때 다른 네트워크가 한방향에만 있기때문에
어떤 주소를 받든간에 같은곳으로 보낼 것이기 때문에 목적지 주소를 디폴트주소로 입력할 수 있다.

ex)
가장자리 라우터  --- > 연결된 라우터 1.1.1.2
에서 static 설정할때

디폴트 주소 0.0.0.0
서브넷 마스크 0.0.0.0
next hop 1.1.1.2
로 하면 끝이다.
일일이 주소 넣어줄 필요없음.
