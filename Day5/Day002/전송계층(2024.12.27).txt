4 전송 계층
역할 : 종단간 데이터의 신뢰성 보장
목적 :End to End = 송,수신간(양 끝단)의 데이터가 잘 도착했는지 확인하고 검수하는 역할이다.

프로토콜 
/////중요!!!///
TCP
- 연결 지향 Protocol = 필요할때마다 연결 가능
데이터를 지속적으로 보내기 위해서 신뢰도를 확보하기 위한 역할( = 3-way handshake)
순서 보장, 재전송가능, 흐름 제어, 오류 제어.
Ex) 웹 브라우징(http/https), 이메일(smtp), 파일전송(FTP)

UDP 
- 비연결지향 Protocol

TCP보다 구조가 단순=> 속도가 빠름 => 실시간thd  응용에 적합
ex) 실시간 스트리밍, 온라인 게임, VolP.



//////////////////문제/////////////////
1 OSI 모델중 4계층 모델이 하는 역할에 대해서 서술하세요
역할 : 종단간 데이터의 신뢰성을 보장하기위한 역할
목적 : End to End 송수신간 데이터가 잘 도착했는지 확인 및 검수.

2 4 계층 관련 프로토콜은 어떤게 있고 각각 특징을 서술하세요

1)TCP
연결 지향 Protocol이며,
3- way handshake를 통해 데이터를 지속적으로 보내기 위해 신뢰도를 확보한다.
신뢰도 확보로 순서보장, 재전송 가능, 흐름 제어, 오류 제어
웹 브라우징, 이메일, 파일전송에 주로 사용된다.

2)UDP
비연결 지향 Protocol이며,
TCP에 비해 구조가 간단하여 속도가 빨르지만 신뢰성을 보장하지 않는다.
주로
실시간 스트리밍, 온라인 게임에 사용된다.
