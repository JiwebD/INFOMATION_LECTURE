IPv4 기본   3계층 논리적 주소

BIT/BYTE
Bit : 비트(Binary Digit)  아라비아숫자(0-9)를 2진수(Binary)로 변환 표현할때 표현되는 한자리
Byte(또는 Octet옥텟) : 8Bit의 1묶음 00000000


1bit : 2^1 = 2(0-1)
2bit : 2^2 = 4(0-3)
3bit : 2^3 = 8(0-7)
4bit : 2^4 = 16(0-15)
	.
	.
	.


NETWORK IP / HOST IP

ex)  아파트 201동/202동/203동  = NETWORK IP 
	    00호/00호 /00호    = HOST IP

ex)  A+1, A+2, A+3/  B+1, B+2, B+3 라는 6가지네트워크는 A/B 영역으로 나뉜다.

A 영역 => A 넷IP		/	B 영역 => B 넷IP  
     
ex) A영역에서 B+1 넷IP를 요청하면  A 영역에서 벗어나 B 영역에서 찾는다.


HOST IP
= 장치마다 각각 host 식별 ip


SUBNET MASK
NETWORK/호스트ip를 지정하기 위해 사용되는 마스크
32bit로 구성
1 : 네트워크 IP(Prefix)
0 : 호스트 자리(UnPrefix)

11000000.168.10.1                
11111111.00000000.0.0  - 서브넷마스크
11000000.168.0.0 = 네트워크 ip
11000000.0.0.1 - 11000000.255.255.254  = 호스트ip범위  
////255 x 254 O 마지막 자리수는 쓰지 않는다

192.168.10.1
255.255.0.0     - 서브넷마스크
192.168.0.0  네트워크ip
192.168.0.1 - 1902.168.255.254   = 호스트ip 범위


클래스풀
인터넷 상 ip 주소를 규결화된 크기별(클래스별)로 구분시키는 방식
A클래스		B클래스		C클래스

255.0.0.0		255.255.0.0	255.255.255.0
/8bit		/16		/24


IPv4 통신방식
Unicast 
1:1방식 => 정확함 => 비효율

Broadcast
1:다수
=> 빠름 => 보안에 치명적

Multicast
1:원하는 특정 다수
= unicast+Broadcast
